How it works?

  • Select whether you want to perform a Standard or Advanced install. 1
  • Follow the instructions using the installation wizard. 2
  • Click on the link to open the CloudFormation console with your pre-generated stack parameters. 3

Let SOCA install everything



Pre-Requisites
You must have built and uploaded your Scale-Out Computing on AWS cluster on S3 bucket.
You must have awscli installed and configured

I want to use my existing resources


Pre-Requisites
You must have built and uploaded your Scale-Out Computing on AWS cluster on S3 bucket.
You must have awscli installed and configured
You must have a VPC
Your VPC must have 6 subnets (3 private/3 public)
You must have 2 Elastic File Systems
You must have one SSH keypair
You must have two security groups configured correctly
What resources SOCA will still create:
  • The EC2 instance used as Scheduler machine
  • 3 IAM roles
  • 1 Lambda function
  • 1 Elastic IP (EIP)

IAM Permissions

This tool only perform "ReadOnly" operations. Here is the list of API permissions your IAM user must have in order to be able to use this tool.

For Standard Installation

{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Sid": "WebBasedSOCAPermissions",
            "Effect": "Allow",
            "Action": [
                "s3:ListAllMyBuckets",
                "s3:ListBucket",
                "ec2:DescribeKeyPairs",
                "ec2:DescribeRegions"
            ],
            "Resource": "*"
        }
    ]
}
For Advanced Installation

{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Sid": "WebBasedSOCAAdvancedPermissions1",
            "Effect": "Allow",
            "Action": [
                "s3:ListAllMyBuckets",

                "ec2:DescribeSubnets",

                "ec2:DescribeRegions",

            ],
            "Resource": "*"
        },
        {
            "Sid": "WebBasedSOCAAdvancedPermissions2",
            "Effect": "Allow",
            "Action": [
                "elasticfilesystem:DescribeMountTargets",

                "elasticfilesystem:DescribeMountTargetSecurityGroups"
            ],
            "Resource": "arn:aws:elasticfilesystem:*:*:file-system/*"
        }
